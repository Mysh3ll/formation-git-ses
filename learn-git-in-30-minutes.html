1. Les bases

Git est une collection d'utilitaires de ligne de commande qui suivent et enregistrent les modifications dans les fichiers (le plus souvent le code source, mais vous pouvez suivre tout ce que vous souhaitez). Avec lui, vous pouvez restaurer les anciennes versions de votre projet, comparer, analyser, fusionner les modifications et plus encore. Ce processus est appelé contrôle de version. Il existe un certain nombre de systèmes de contrôle de version qui font ce travail. Vous avez peut-être entendu certains d'entre eux - SVN, Mercurial, Perforce, CVS, Bitkeeper et plus encore.

Git est décentralisé, ce qui signifie qu'il ne dépend pas d'un serveur central pour conserver les anciennes versions de vos fichiers. Au lieu de cela, il fonctionne entièrement localement en stockant ces données sous forme de dossier sur votre disque dur, que nous appelons un dépôt. Cependant, vous pouvez stocker une copie de votre dépôt en ligne, ce qui permet à plusieurs personnes de collaborer et de travailler facilement sur le même code. C'est à cela que servent des sites Web comme GitHub et BitBucket.

1. Installer Git

L'installation de Git sur votre machine est simple:
  • ➔ Linux - Ouvrez simplement un nouveau terminal et installez git via le gestionnaire de paquets de votre distribution. Pour Ubuntu, la commande est: sudo apt-get install git
  • ➔ Windows - nous recommandons git pour windows car il offre à la fois un client GUI et un émulateur de ligne de commande BASH.
  • ➔ OS X - La façon la plus simple est d'installer homebrew, puis d'exécuter brew install git depuis votre terminal.

Si vous êtes un débutant absolu, un client graphique git est un must. Nous recommandons TortoiseGit, mais il existe de nombreux autres bons et gratuits en ligne. Connaître les commandes de base de git est toujours important même si vous utilisez une application GUI, donc pour le reste de cette leçon, ce sera notre seul objectif.

2. Configurer Git

Maintenant que nous avons installé git sur notre ordinateur, nous devrons ajouter quelques configurations rapides. Il existe de nombreuses options qui peuvent être manipulées, mais nous allons configurer les plus importantes: notre nom d'utilisateur et notre adresse e-mail. Ouvrez un terminal et exécutez ces commandes: 

					
						# Configuration du nom d'utilisateur
						git config --global user.name "My Name"

						# Configuration de l'adresse e-mail
						git config --global user.email myEmail@example.com

Chaque action que nous faisons dans Git aura désormais un tampon avec notre nom et notre adresse. De cette façon, les utilisateurs savent toujours qui a fait quoi et tout est beaucoup plus organisé.
